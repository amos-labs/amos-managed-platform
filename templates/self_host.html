Self-Host AMOS

Run your own AMOS harness on your infrastructure. Free forever, Apache 2.0 licensed.

Quick Start

Pull the image and run with Docker. Requires PostgreSQL and Redis.

# Pull the latest AMOS harness image
docker pull ghcr.io/amos-labs/amos-harness:latest
# Run with required environment variables
docker run -d --name amos-harness \
-p 3000:3000 \
-e AMOS__DATABASE__URL=postgres://user:pass@host:5432/amos \
-e AMOS__REDIS__URL=redis://host:6379 \
-e AWS_PROFILE=your-profile \
ghcr.io/amos-labs/amos-harness:latest

macOS (Docker Desktop)

1. Install Docker Desktop

Download from docker.com or install via Homebrew:

brew install --cask docker

2. Start infrastructure

Run PostgreSQL and Redis:

docker run -d --name amos-postgres \
-e POSTGRES_USER=amos -e POSTGRES_PASSWORD=amos -e POSTGRES_DB=amos \
-p 5432:5432 postgres:16
docker run -d --name amos-redis -p 6379:6379 redis:7-alpine

3. Run the harness

docker run -d --name amos-harness \
-p 3000:3000 \
-e AMOS__DATABASE__URL=postgres://amos:amos@host.docker.internal:5432/amos \
-e AMOS__REDIS__URL=redis://host.docker.internal:6379 \
-e AWS_PROFILE=default \
ghcr.io/amos-labs/amos-harness:latest

4. Open in browser

Visit http://localhost:3000 to access your harness.

Windows & Linux (Docker)

1. Install Docker

Windows: Install Docker Desktop for Windows (requires WSL2).

Linux: Install Docker Engine via your package manager.

# Ubuntu/Debian
curl -fsSL https://get.docker.com | sh

2. Use Docker Compose (recommended)

Save this as docker-compose.yml and run docker compose up -d:

services: postgres: image: postgres:16 environment: POSTGRES_USER: amos POSTGRES_PASSWORD: amos POSTGRES_DB: amos ports: ["5432:5432"] volumes: ["pgdata:/var/lib/postgresql/data"] redis: image: redis:7-alpine ports: ["6379:6379"] amos-harness: image: ghcr.io/amos-labs/amos-harness:latest ports: ["3000:3000"] environment: AMOS__DATABASE__URL: postgres://amos:amos@postgres:5432/amos AMOS__REDIS__URL: redis://redis:6379 depends_on: [postgres, redis] volumes: pgdata:

CLI (amos-cli)

Install the AMOS CLI

The CLI can manage local harness instances, run migrations, and configure your environment.

# Install via cargo (Rust toolchain required)
cargo install amos-cli
# Or build from source
git clone https://github.com/amos-labs/amos-automate.git
cd amos-automate
cargo build --release --bin amos
./target/release/amos --help

Setup and run

# Initialize local environment
./scripts/dev-setup.sh
# Run the harness
cargo run --bin amos-harness

Requirements

Infrastructure

  • PostgreSQL 15+ (with pgvector extension)
  • Redis 7+
  • 2+ GB RAM
  • 2+ CPU cores

AI Provider

  • AWS account with Bedrock access (Claude)
  • Or bring your own API key (BYOK)
  • Configure via the harness Settings page
